**영역별 진단**

- **backend 분석 파이프라인 (B)** — 가장 완성도 높음. 수집(AN-01)·6항목 판별(AN-02)·STEP2 리포트(AN-03/04)가 실제 동작. 남은 공백은 인증·DB 레이어·HTTP 엔드포인트.
- **frontend (C)** — 전 페이지 UI·컴포넌트 완성. 다만 데이터 공급 함수가 대부분 MOCK이며 인증·온보딩 화면은 파일 부재. backend 연결과 DB 도입이 핵심 과제.
- **agent (C)** — 스케줄러·신규 공시 감지·파이프라인 브리지·Solar 채팅이 각각 동작하나 분절돼 있음. **D-03의 trigger→pipeline 연결**이 핵심 미연결 지점 — PR #20에서 monitoring skill로 해소 예정.
- **공통 블로커** — `backend/db/`(또는 Supabase) persistence 부재로 reports·qa_history·agent_run_logs·watchlist·user 테이블이 전부 없음. AUTH·R-01/02·D-04·PF-02 등 DB 의존 기능이 모두 여기서 막힘.
